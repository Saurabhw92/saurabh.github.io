GitLab CI/CD and Terraform

What is the project about?

This project focuses on automating the infrastructure provisioning and deployment process for hosting a React application on AWS. Here's an overview:

Infrastructure as Code (IaC) with Terraform: Terraform was used to define and provision an Amazon S3 bucket, which serves as the hosting platform for the React application. Using Terraform ensures consistent and repeatable infrastructure creation.

GitLab CI/CD Pipelines: GitLab CI/CD pipelines were employed to automate the deployment process. This involves stages such as building the React application, syncing the application files to the S3 bucket, and ensuring the application is deployed seamlessly.

State Management with GitLab: GitLab was also used to maintain the Terraform state file, ensuring version control and team collaboration for infrastructure management.

Application Hosting: The S3 bucket is configured to serve the React application as a static website, making it accessible to end-users via the internet.

Technology stack

  • React
  • GitLab CI/CD
  • Terraform
  • AWS S3
