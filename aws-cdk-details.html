GitLab CI/CD and AWS Cloud Development Kit (CDK)

What is the project about?

This project involves several key components that work together to automate the deployment of a React application to AWS. The first component is the React Application Setup, which includes installing Node.js and npm, creating a new React application using the create-react-app.

The second component is the GitLab Repository Configuration. This involves creating a new repository on GitLab to host the React application code.

The third component is the AWS CDK Project Setup. AWS Cloud Development Kit (CDK) is used to define and provision the cloud infrastructure required for the project. This involves installing AWS CDK, initializing a new CDK project with TypeScript, and installing necessary CDK libraries for S3 and S3 deployment. The infrastructure is defined in the CDK stack, which includes creating an S3 bucket to host the React application and configuring the deployment settings. This component allows for infrastructure as code, making the deployment process repeatable and manageable.

The fourth component is the GitLab CI/CD Pipeline Configuration. A .gitlab-ci.yml file is created to define the CI/CD pipeline stages, which include build and deploy stages. The build stage installs dependencies and builds the React application, while the deploy stage installs AWS CDK, navigates to the CDK project directory, installs dependencies, and deploys the infrastructure and application to AWS. This pipeline automates the process of building and deploying the application, ensuring that any changes pushed to the GitLab repository are automatically deployed to AWS.

Once the deployment is complete, the application can be accessed via the S3 bucket URL.

Technology stack

  • React
  • GitLab CI/CD
  • AWS CDK
  • AWS S3
