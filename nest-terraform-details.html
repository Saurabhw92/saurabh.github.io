NestJs with GitLab CI/CD and Terraform

What is the project about?

This project focused on deploying a NestJS application using modern cloud technologies and DevOps tools. The application was containerized using Docker, with the image stored in AWS ECR (Elastic Container Registry). Using GitLab CI/CD, the image was built and pushed to ECR, automating the deployment process. Terraform was employed for infrastructure provisioning, including AWS ECS (Elastic Container Service) clusters, load balancers, and security groups. The ECS cluster was integrated with a Load Balancer to route internet traffic securely. The AWS CLI was leveraged to trigger ECS deployments automatically whenever a new image was pushed to the registry.

Technology stack

  • Terraform
  • Docker
  • GitLab CI/CD
  • AWS ECS (Elastic Container Service)
  • AWS ECR (Elastic Container Registry)
  • AWS Application Load Balancer
  • AWS CLI
